# Self-Improvement Protocol — рівень фабрики (успадковано з grow-pm)

Застосовується: (а) наприкінці будь-якого skill після корекцій Andrii; (б) систематично — у L6 Retro Loop.

## Тригер
Andrii виправив вихід / вказав на помилку / попросив змінити структуру. НЕ тригериться на «ок, все добре» чи разові контентні правки.

## Кроки
1. **Виправити** те, що просили. Підтвердження.
2. **Root cause:** що пішло не так → чому (прогалина інструкції? відсутній крок? неврахований кейс?) → чи це ПАТЕРН чи one-off → скоуп (один skill / кілька / паспорт петлі / конституція).
3. **Пропозиція** (тільки якщо патерн): конкретна мінімальна зміна — «поточна поведінка → пропонована → як запобігає». Кілька — нумерованим списком на вибір.
4. **Застосування** (тільки з дозволу Andrii): правка файлу → bump версії skill (PATCH/MINOR/MAJOR) → bump plugin.json за найвищим імпактом → запис у CHANGELOG.md плагіну → якщо змінено процес фабрики — запис у strw-state/process-changelog.md.

## Правила
- Мінімальна зміна, що закриває патерн. Без over-engineering.
- Чесність: one-off (унікальний контекст, смак користувача) — так і сказати, нічого не міняти.
- Зміни паспортів петель = MINOR мінімум; зміна структури петлі = MAJOR.
