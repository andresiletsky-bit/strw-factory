# Budget Policy — бюджет як проєктне обмеження

## Стелі (джерело правди: strw-state/budget.md)
- Підписка Claude + max $200/міс зовнішніх витрат.
- ≤6 паралельних субагентів на будь-який fan-out.
- Ліміти запусків/тиждень на петлю — таблиця в budget.md.

## Політика моделей
| Робота | Модель |
|--------|--------|
| Triage, дедуп, фільтри, перевірка схем полів | дешева (haiku-клас) |
| Maker критичних артефактів (validation-report, PRD, код) | сильна |
| Checker критичних артефактів | **HARD RULE: ІНША модель, ніж maker** (дефолт: maker opus → checker sonnet) + інший промпт. Виняток (інша модель недоступна) — фіксується в loops-log з приміткою |
| Структурна перевірка артефактів | БЕЗ моделі — `scripts/validate-artifact.sh` (детермінований рівень 0) |
| Скани/читання великих обсягів | дешева, через субагентів (компактні структури назад) |

## Правила виконання
1. Кожен запуск петлі ПОЧИНАЄТЬСЯ з budget check і ЗАКІНЧУЄТЬСЯ записом факту в budget.md.
2. Стеля вичерпана → STOP + `budget-alert` у triage-inbox. Ніколи не «доробити тихенько».
3. Будь-яка ЗОВНІШНЯ витрата грошей (API-ключі, хостинг, домен, реклама) — незворотна дія → тільки через gate-запит в inbox, виконує Andrii.
4. Групування: headless-запуск виконує всі належні дії дня одним заходом (напр., L5+L6 у п'ятницю).
5. L6 Retro щомісяця переглядає співвідношення «вартість петлі / цінність виходу» і пропонує калібрування кадансів.
