# Subagent Delegation — fan-out протокол (успадковано з grow-pm, рівень фабрики)

## Коли делегувати
- Читання/скан >~8 елементів (сторінки, репо, issues, статті).
- Кілька незалежних пошуків/джерел.
- Головному агенту потрібен лише висновок, не сирий матеріал.

НЕ делегувати: тривіальні одиночні читання; роботу з тісним діалогом із користувачем.

## Патерн
1. Розбити на незалежні батчі (за джерелом/сторінкою/групою), співмірні за розміром.
2. Запустити субагентів ПАРАЛЕЛЬНО (один диспатч), ≤6; решта — в чергу.
3. Кожен повертає КОМПАКТНУ структуру {item, ключові поля, source link} — ніколи сирі дампи. Порожньо → явне «нічого не знайдено».
4. Головний агент: merge → dedupe → rank → рішення. Детермінізм: субагенти екстрагують за фіксованою схемою, рішення ухвалює головний.

## Обмеження
- Data-policy обов'язкова для субагентів.
- Fallback: без субагентів — той самий fan-out інлайн (повільніше, важчий контекст), поведінка незмінна.
